# Dashboard Scenario Dataset

## Purpose

The dashboard seed is a deterministic manager-side RFQ portfolio for executive and operational dashboard testing. It extends the curated `must-have + later` scenario set without changing those existing batches.

Use it when the UI, copilot, or analytics work needs meaningful distributions across status, clients, owners, stages, blockers, urgency, and outcomes.

## Dataset Size

- Public seed batch: `dashboard`
- Total seeded RFQs: 120
- Existing baseline: 34 RFQs from `must-have + later`
- Dashboard extension scenarios: 86 RFQs
- Extension scenario keys: `DASH-D001` through `DASH-D086`
- Runtime RFQ codes: still generated by the IF/IB counter logic
- Manual reservation: `RFQ-06` remains unseeded

## Intentional Dashboard Patterns

- Client concentration: repeated strategic clients create useful top-client and comparison charts.
- Owner workload: assignments are intentionally imbalanced across existing proposal owners.
- Stage bottlenecks: active RFQs are spread across intake, clarification, design, BOQ/BOM, vendor inquiry, estimation, approval, submission, and decision-wait stages.
- Urgency: overdue, due-soon, and on-track RFQs are all represented.
- Blockers: active blockers use meaningful reason codes such as `waiting_client_docs`, `vendor_quote_delay`, `internal_approval_hold`, and `commercial_exception`.
- Outcomes: awarded, lost, and cancelled RFQs include outcome reasons and terminal state consistency.
- Readiness markers: source/workbook file defaults and intelligence profiles are reused from existing scenario conventions.

## Run

From `microservices/rfq_manager_ms`:

```bash
python scripts/seed_rfqmgmt_scenarios.py --batch dashboard
```

For the full local manager + intelligence scenario stack:

```bash
python ../../scripts/rfqmgmt_scenario_stack.py all --seed-set full-plus-dashboard
```

## Verify

```bash
python -m pytest tests/unit/test_rfqmgmt_scenario_seed.py tests/unit/test_seed_runtime_sync.py tests/unit/test_rfq_datasource_analytics.py -q
python ../../scripts/rfqmgmt_scenario_stack.py verify --seed-set full-plus-dashboard
```
